Unlike the security features available in most public clouds, the highly isolated, turn-key Oracle Cloud Infrastructure (OCI) Security starts at the firmware tier. Branching across different cloud components, OCI security features guarantee maximum, round-the-clock security and control of enterprise workloads against ever-evolving threats.

Oracle Cloud Infrastructure Security Features

  • Cross-tenant threat containment: Based on the 2nd generation cloud, OCI does not allow cross-tenant threat access via segregated hypervisor and server/network virtualization.
  • Malware-resistant hardware (Root of trust): Security in OCI is enforced from the hardware’s firmware and BIOS tier, preventing firmware-level threats.
  • Least trust (or Zero trust) design: Servers, hypervisors, and tenants do not trust each other by default.

  • Customer isolation: Deploy applications and data assets inside a fully isolated environment, inaccessible by other workloads or tenants.
  • Discovery of security weakness: Get a clear picture of weak points in resource configurations and risky user actions.
  • Fault-independent data centers: Ensure highly available, scale-out architectures resilient against cyber-attacks and disasters.

  • Always-on encryption: The protection of customer data in OCI is always on, whether at-rest or in-transit.
  • Security zones: A set of policies in OCI that guarantees compliance of your resources with industry best practices.
  • Admin-friendly IAM policies: Regulate access to services while segregating operational responsibilities to minimize risks.

Oracle Cloud Guard provides an integrated view of your OCI security posture, detecting misconfigurations and suspicious activities with automated remediation. This helps scale security operations and proactively address threats.

Data Safe helps customers identify data sensitivity, gauge risks, mask sensitive data, enforce security controls, and ensure data security compliance.
